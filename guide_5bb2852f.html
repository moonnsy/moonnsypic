ПОДКЛЮЧЕНИЕ SPOTIFY В MOODTUBE

Подключение Spotify в MoodTube

Содержание

  • 1. Создание приложения Spotify
  • 2. Авторизация в таверне
  • 3. Прикручиваем бесплатный HTTPS (домен) для серверов

1 Создание приложения Spotify

Для того чтобы MoodTube мог управлять музыкой, нужно получить специальные ключи доступа к вашему аккаунту Spotify.

  1. Перейдите на сайт для разработчиков: developer.spotify.com/dashboard и войдите в свой аккаунт.
  2. В открывшемся Dashboard найдите и нажмите кнопку Create App.
  3. Заполните базовые поля:
    • App name: придумайте любое название (например, TavernMoodTube)
    • App description: любое описание (например, My music player)
    • Redirect URLs:
      Здесь нужно указать адрес вашей Таверны. Если вы используете её на телефоне или локально, впишите:
      http://127.0.0.1:8000/
      💡 Подсказка: само расширение MoodTube покажет точный адрес, который нужно сюда вписать.
  4. Обязательно поставьте галочку напротив WebAPI и завершите создание кнопкой Create.

✨ Готово! Теперь зайдите в настройки созданного приложения, скопируйте свой Client ID, вставьте его в настройки расширения MoodTube в Таверне и нажмите кнопку Авторизоваться.

2 Авторизация в таверне

Теперь нужно связать расширение с вашим новым приложением Spotify.

  1. Откройте настройки MoodTube в Таверне.
  2. Перейдите во вкладку API Настройки.
  3. В Источнике воспроизведения выберите Spotify.
  4. Введите свой ClientID, который мы получили на первом шаге.
  5. Нажмите Авторизоваться.
  6. Вас перебросит на авторизацию в Spotify, войдите и нажмите Разрешить.
  7. Далее дождитесь появления зелёной надписи "Авторизация успешна! Можете закрыть эту вкладку" (она в целом закроется сама).
  8. После обязательно нажмите Сохранить внизу настроек MoodTube.

3 Прикручиваем бесплатный HTTPS (домен) для серверов

Если вы используете для таверны свой сервер и не получается подключить Spotify с обычным IP (мобильные браузеры могут блокировать кнопку копирования, а Spotify-расширение отказывается авторизовываться), то вам нужен HTTPS!

Всё это происходит из-за отсутствия защищенного соединения. Сервисы вроде Cloudflare Tunnels хороши, но если использовать их без регистрации, ссылка меняется при каждом рестарте сервера. Ниже — пошаговая инструкция, как один раз и навсегда настроить бесплатный домен с автоматическим продлением SSL-сертификата с помощью DuckDNS, Nginx и Let's Encrypt.

3.1 Получаем бесплатный домен

Сначала забронируем красивое имя для вашего сервера.

  1. Перейдите на сайт duckdns.org и авторизуйтесь (через Google, GitHub и т.д.).
  2. В блоке Domains введите желаемое имя латиницей без пробелов (например, my-best-tavern) и нажмите add domain.
  3. В появившемся списке найдите ваш новый домен.
  4. В поле current ip сотрите то, что там написано, и впишите IP-адрес вашего сервера (виртуальной машины). Нажмите update ip.

3.2 Установка Nginx и Certbot на сервер

Зайдите в консоль вашего сервера (через SSH или Termux). Нам нужно установить веб-сервер и робота, который будет сам получать и обновлять сертификаты.

Обновляем списки пакетов:

Bash
apt update

Устанавливаем Nginx и Certbot:

Bash
apt install nginx certbot python3-certbot-nginx -y

3.3 Настраиваем перенаправление трафика (Reverse Proxy)

Теперь нужно объяснить Nginx, что весь трафик с вашего нового домена нужно перенаправлять в Таверну (по умолчанию она работает на порту 8000).

Создаем конфигурационный файл. Вместо ваше_имя напишите название, которое будет удобно вам (например, название домена):

Bash
nano /etc/nginx/sites-available/ваше_имя

Откроется редактор. Вставьте туда следующий код (обязательно замените твой_домен.duckdns.org на тот, что вы зарегистрировали в шаге 3.1).
Обратите внимание: строка client_max_body_size 50M; очень важна — она позволяет сохранять тяжелые настройки Таверны без ошибки 413.

Nginx
server { listen 80; server_name твой_домен.duckdns.org; client_max_body_size 50M; location / { proxy_pass http://127.0.0.1:8000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade"; } }

Чтобы сохранить файл в редакторе nano, нажмите Ctrl + O, затем Enter, а для выхода — Ctrl + X.

3.4 Активация конфигурации

Создаем ярлык файла в папке активных сайтов (не забудьте подставить ваше_имя из предыдущего шага):

Bash
ln -s /etc/nginx/sites-available/ваше_имя /etc/nginx/sites-enabled/

Проверяем, нет ли ошибок в коде:

Bash
nginx -t

Если в ответ вы получили сообщение syntax is ok, перезагружаем Nginx:

Bash
systemctl restart nginx

3.5 Получение сертификата (Магия Let's Encrypt)

Запускаем процесс получения сертификата. Подставьте ваш домен:

Bash
certbot --nginx -d твой_домен.duckdns.org

Робот задаст пару вопросов прямо в консоли:

  • Введите свой email (туда придет письмо, если сертификат вдруг не обновится).
  • Согласитесь с правилами (введите Y и нажмите Enter).
  • На предложение подписаться на рассылку откажитесь (введите N и нажмите Enter).
